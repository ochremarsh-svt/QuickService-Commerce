1. Template Overview

Welcome to the official BaryaBites documentation portal. BaryaBites is a premium, smart pocket-sized micro-menu web template ideal for small-scale culinary operations, fast food vendors, or plant-based pop-ups. It features modern performance modules, fluid micro-interactions, responsive category filtering, and a powerful decoupled shopping cart architecture.

Designed with exceptional focus on responsive constraints, clean UI spacing, and low code overhead ideal for premium theme asset reviewers.

2. Folder & File Structure

The clean layout organizes your system files cleanly to maintain clarity, optimize modular tracking, and ensure hassle-free marketplace validation.

3. Dynamic Product Routing Engine

Rather than forcing developers to copy, paste, and modify multiple physical static files for every singular menu entry, BaryaBites utilizes an advanced URL Parameter Router Matrix. This ensures clean navigation vectors and eliminates code bloat entirely.

How Card Click Routing Works

Each card item element in the primary catalog grid utilizes clean inline window navigation parameters to supply data vectors directly into the detail template layer: 

<!-- Production Grid Dynamic Card Setup Example -->
<div class="menu-card" data-category="burgers" onclick="window.location.href='item-details.html?id=bb-02'">
    <div class="product-img" style="background: url('assets/img/pita.jpg') center/cover;"></div>
    <div class="card-details">
        <h3>Dynamic Falafel Pita</h3>
        <div class="card-footer">
            <span class="price">$4.50</span>
        </div>
    </div>
</div>

When loaded, item-details.html immediately decodes the parameter matrix to display relevant metadata variables from its embedded local data object lookup framework.

4. Dynamic Price Recalculation

Inside the layout engine of item-details.html, price and modifier states calculate customer choices dynamically based on checkbox input interactions. Total costs multiply transparently alongside real-time quantity state hooks. 

// Dynamic Recalculation Segment Logic
function recalculateTotals() {
    let addonTotal = 0;
    checkboxes.forEach(box => {
        if (box.checked) {
            addonTotal += parseFloat(box.getAttribute('data-addon-price'));
        }
    });

    const unitCost = basePrice + addonTotal;
    const grandTotal = unitCost * currentQuantity;

    priceDisplay.textContent = `$${unitCost.toFixed(2)}`;
    projectionDisplay.textContent = `$${grandTotal.toFixed(2)}`;
}

5. Cart Integration Engine

The client cart system relies on structured line-item payload data states. When custom variants pass into your local cart database module, metadata string descriptors assemble flawlessly, preventing data collisions between distinct modifier configurations.

Structured Payload Sample

{
    id: "bb-01-custom-1718301213000",
    name: "Classic Smash Burger (Extra Cheddar Cheese Slice)",
    price: 7.49,
    quantity: 2
}

This schema tracks quantities, modifications, and exact cost records accurately across multiple view panels throughout the lifetime of the checkout tracking session.
